PRODUCT REQUIREMENTS DOCUMENT (PRD)
RoomMate Manager - Comprehensive Shared Living Management Platform

================================================================================
PROJECT OVERVIEW
================================================================================

Product Name: RoomMate Manager
Version: 1.0 MVP
Target Users: Students, Working Professionals, Bachelors in shared living (India, Nigeria, UAE, Kuwait, Global)
Tech Stack: React, Supabase (PostgreSQL), Cloudinary (Image Storage)
Monetization: Ad Revenue Sharing (Google AdMob/AdSense)
Primary Goal: Stress-free management of shared living expenses, meals, tasks, and resources

================================================================================
CORE FEATURES & SPECIFICATIONS
================================================================================

1. AUTHENTICATION & USER MANAGEMENT
================================================================================

1.1 User Registration & Onboarding
- Email/Password authentication via Supabase Auth
- Multi-step onboarding wizard:
  * Personal Information: Name, Age, Phone, Email
  * Identity Verification: Upload ID Card (Cloudinary), Resume/CV, Labour Card
  * Profile Photo upload (Cloudinary with face detection validation)
  * Emergency Contact Information
- Admin approval required before account activation
- Unique user ID generation with QR code for identification

1.2 User Roles & Permissions
- Super Admin: Full system access, apartment creation, billing configuration
- Admin: Apartment/room management, user approval, payment verification
- Roommate (Full): Access to all features (rent, bills, meals, tasks)
- Roommate (Partial): Custom feature access (e.g., only community meals)
- Guest: Temporary access with limited features
- Outsider: Community meal only, no apartment access

1.3 User Profile Management
- Private profile visible only within apartment network
- Profile fields:
  * Basic info (name, age, contact)
  * Documents (ID, resume, labour card)
  * Current location: Apartment > Room > Bed Type
  * Loyalty Points Score (visible to all in apartment)
  * Payment history and status
  * Task completion rate
  * Community meal participation status
  * Linked relationships (if related to other roommates)

================================================================================
2. APARTMENT & ROOM MANAGEMENT
================================================================================

2.1 Hierarchical Structure
- Multi-level folder system: Apartment → Room → Bed
- Apartment naming: Custom names (e.g., "Greg's Apartment")
- Room naming: Custom or numbered (e.g., "Hailey's Room", "Room 305")
- Bed types with pricing tiers:
  * Flat Bed (Premium - Highest price)
  * Lower Bed (Standard - Medium price)
  * Upper Bed (Economy - Lowest price)

2.2 Apartment Setup & Configuration
- Create multiple apartments (Free: 2 apartments, Paid: Unlimited)
- Per apartment configuration:
  * Total monthly rent
  * Room-wise rent allocation
  * Bed-wise pricing
  * Shared bills (Electricity, Internet, Gas, Water)
  * Custom bill categories
- Visual folder navigation with icons and colors
- Apartment network creation (link 2-3 apartments)

2.3 Room & Bed Allocation
- Drag-and-drop bed assignment to users
- Visual bed layout designer (arrange beds in room view)
- Real-time vacancy tracking
- Red indicator for vacant beds (visible to all members)
- Bed swap functionality with admin approval
- Guest bed booking (temporary allocation)

2.4 Vacancy Management & Notifications
- Automatic vacancy alerts when user exits
- Broadcast vacancy to all apartment members
- Invite link generation for new roommate recruitment
- QR code sharing for apartment joining
- Roommate referral tracking
- Exit approval workflow:
  * User requests exit
  * Admin verifies all payments cleared
  * Final settlement calculation
  * Loyalty points settlement
  * Exit confirmation

  * Maintenance/Cleaning (shared)

- Bill splitting logic:

3.2 Payment Tracking System (Manual - MVP)

- Automatic calculation based on:

  * Pro-rated calculations for mid-month join/exit
- User payment dashboard showing:

  * Due date

3.3 Payment Reminders & Penalties

- Payment streak tracking (on-time payment count)

3.4 Annual Payment Incentives
- Payment frequency options: Monthly/Quarterly/Annual
- Discount calculator:

- Savings dashboard showing money saved
- Automatic pro-rated refund for early exit

3.5 Payment Reports & Analytics

  * Performance rating system

4.2 User Participation Options

  * Lunch only
  * Dinner only

  * Per-meal pricing calculation
- Drinking water inclusion toggle
- Opt-in/Opt-out functionality:
  * Requires admin approval for opt-in
  * 1-week notice for opt-out
  * Pro-rated billing adjustments

4.3 Weekly Menu Management
- 7-day menu planner
- Meal composition categories:
  * Main Course: Rice, Roti, Porotta, etc.
  * Curry 1, Curry 2
  * Side dishes
  * Dessert
  * Beverages
- Menu items CRUD:
  * Admin adds/edits/deletes menu items

  * Allergen warnings
- Voting & Polls for menu:
  * Admin creates food polls
  * Users vote on menu additions
  * Top-voted items added to rotation
  * Voting history tracking

4.4 Grocery Team Rotation System
- Automatic team generation:
  * Equal distribution algorithm
  * Example: 17 users → 7 teams of 2 + 1 team of 3
  * Example: 18 users → 9 teams of 2
- 4-week rotation cycle
- Team assignment display:
  * Week 1: Team A
  * Week 2: Team B
  * Week 3: Team C
  * Week 4: Team D
  * (Cycle repeats)

- Admin can manually reassign teams

4.5 Grocery Bill Management
- Team grocery purchase tracking:
  * Manual bill entry with itemized list
  * OCR receipt scanning (Cloudinary + OCR API):
    - Upload receipt photo
    - Auto-extract items, quantities, prices
    - Edit extracted data before saving
    - Attach original receipt image
  * Bill categories:
    - Groceries
    - Gas cylinder
    - Drinking water
- Bill breakdown:
  * Item name, quantity, price
  * Total amount
  * Purchase date
  * Store name
  * Purchased by (team members)
- Admin review dashboard:
  * View all team purchases
  * Weekly spend comparison
  * Item-wise purchase history
  * Approve/flag suspicious purchases

4.6 Cost Splitting Logic
- Fixed budget model (Optional):
  * Admin sets fixed amount per person (e.g., ₹400/week)
  * If team spends more: Team pays difference
  * If team spends less: Refund distributed to all users
- Dynamic split model:
  * Total weekly cost divided by participant count
  * Different rates for:
    - Full meals users
    - Lunch-only users (50% of full price)
    - Dinner-only users (50% of full price)
    - No-water users (deduct water cost)
- Pro-rated calculations for mid-week join/exit

4.7 Team Performance & Feedback
- Rating system for grocery teams:
  * 5-star rating after each week
  * Rating categories:
    - Budget management
    - Quality of items
    - Timeliness
    - Communication
- Feedback comments section
- Team leaderboard (highest rated teams)
- Loyalty points bonus for high-rated teams

4.8 Community Meal Analytics
- Participant dashboard:

  * Savings calculation
  * Favorite meals tracking
- Admin analytics:
  * Average cost per meal
  * Weekly budget trends
  * Most/least popular menu items
  * Team performance comparison

  * Mopping common areas

- Custom task creation by admin
- Task properties:

  * Estimated time duration
  * Difficulty level (Easy/Medium/Hard)
  * Loyalty points reward
  * Frequency (Daily/Weekly/Bi-weekly/Monthly)
  * Preferred time slots

5.2 Task Assignment & Rotation

  * Team tasks (2+ people collaborate)
  * Round-robin rotation
  * Random assignment
- Automatic rotation schedule generation
- Task calendar view (daily/weekly/monthly)
- Task assignment notifications
- Upcoming task reminders (1 day before, 3 hours before)

5.3 Task Exchange System

5.4 Task Completion Tracking
- Task completion workflow:
  * User marks task as "Started"

- Task completion metrics:

  * Quality rating (if peer review enabled)
  * Task completion streak

5.5 Task Penalties & Rewards
- Missed task penalties:

- Task completion rewards:
  * Positive loyalty points
  * Achievement badges
  * Leaderboard ranking
  * Monthly "Roommate of the Month" recognition

5.6 Task Notifications & Reminders
- Multi-channel reminders:
  * In-app notifications
  * Email notifications
  * Optional SMS (future)
- Reminder timing:
  * 1 day before task
  * 3 hours before task
  * 30 minutes before task
  * Overdue alert (if not started)

6. RESOURCE SCHEDULING SYSTEM
================================================================================

6.1 Shared Resource Management
- Default resources:
  * Washing Machine
  * Dryer
  * Iron Box
  * Clothes Drying Area
  * Printer (optional)
- Custom resource creation by admin
- Resource properties:
  * Resource name and photo
  * Total slots per day
  * Slot duration (15/30/60 minutes)
  * Operating hours
  * Max bookings per user per week
  * Advance booking window (e.g., 7 days)

6.2 Slot Booking System
- Calendar-based booking interface
- Visual availability display:
  * Green: Available
  * Yellow: Booked by user
  * Red: Booked by others
  * Gray: Outside operating hours
- Booking workflow:
  * Select resource and date
  * Choose time slot
  * Add notes (optional)
  * Confirm booking
  * Receive confirmation notification
- Booking modifications:
  * Cancel booking (min 1 hour before slot)
  * Reschedule booking
  * Extend slot (if next slot available)

6.3 Slot Usage Tracking
- Check-in/Check-out system:
  * User checks in when starting to use resource
  * User checks out when finished
  * Auto-checkout if user doesn't check out (slot end + 10 min grace)
- No-show tracking:
  * If user doesn't check in within 15 min of slot start
  * Mark as no-show
  * Negative loyalty points penalty
  * 3 no-shows = booking restriction (admin approval required)
- Overstay tracking:
  * If user exceeds booked slot time
  * Grace period: 10 minutes
  * After grace: Penalty charge + negative loyalty points

6.4 Slot Notifications
- Booking confirmation (immediate)
- Reminder 1 hour before slot
- Reminder 15 minutes before slot
- Check-in reminder at slot start time
- Check-out reminder 5 minutes before slot end
- No-show warning after 15 minutes

7. LOYALTY POINTS & GAMIFICATION SYSTEM
================================================================================

7.1 Loyalty Points Structure
- Point earning opportunities:
  * On-time rent payment: +50 points
  * Early payment (5+ days before due): +75 points
  * Task completion on-time: +20 points
  * High-quality task completion (5-star rated): +30 points
  * Emergency task volunteer: +40 points
  * Grocery team high rating (4+ stars): +25 points/member
  * Referral new roommate: +100 points
  * Zero disputes for 3 months: +150 points
  * Watching ad sets (10 ads): +5 points
  * Community meal feedback: +5 points
  * Voting in polls: +2 points
  * Perfect attendance (resource bookings): +10 points/week

7.2 Point Deduction Events
- Late payment (1-3 days): -25 points
- Very late payment (4+ days): -50 points
- Missed task: -30 points
- Poor task quality (2-star or below): -20 points
- Payment dispute (if fault found with user): -40 points
- No-show for resource booking: -15 points
- Overstay resource slot: -10 points
- Raising false complaints: -50 points
- Exit without payment: -500 points (permanent record)

7.3 Loyalty Tiers & Benefits
- Tier structure:
  * Bronze: 0-499 points (Standard benefits)
  * Silver: 500-999 points (+5% discount on fines)
  * Gold: 1000-1999 points (+10% discount, priority booking)
  * Platinum: 2000+ points (+15% discount, 1 free fine waiver/month)
- Tier badges displayed on profile
- Tier benefits:
  * Higher tiers = priority in resource booking conflicts
  * Higher tiers = higher trust score for admin approvals
  * Higher tiers = bonus ad revenue share percentage

7.4 Leaderboards & Achievements
- Multiple leaderboards:
  * Overall loyalty points (all-time)
  * Monthly points leader
  * Task completion champion
  * Payment punctuality leader
  * Apartment-wise rankings
- Achievement badges:
  * "Early Bird" (5 consecutive early payments)
  * "Task Master" (50 tasks completed)
  * "Team Player" (10 emergency task volunteers)
  * "Perfectionist" (20 five-star task ratings)
  * "Reliable" (6 months zero late payments)
  * "Veteran" (1 year in apartment)
  * "Peacemaker" (zero disputes for 6 months)
- Shareable achievements (social media integration - future)

7.5 Points Redemption (Future Phase)
- Redeem points for:
  * Waive late payment fine (500 points)
  * Skip one task (300 points)
  * Priority bed selection when upgrading (1000 points)
  * Extra resource booking slot per week (200 points)

9.1 Pool Creation & Management

- Month 2: User B receives ₹900 (doesn't pay this month)

- Automatic payout schedule generation
- Visual calendar showing:

  * Remaining months

9.3 Pool Participation Tracking
- User dashboard:
  * Total contributed to date
  * Expected payout month

  * Payout status (Received/Pending)
- Admin dashboard:
  * All active pools
  * Contribution collection status
  * Payout completion tracking
  * Defaulters list
- Payment tracking:
  * Manual entry (like regular bills)
  * User marks contribution as paid
  * Admin verifies
  * Automated reminders for missed contributions

9.4 Pool Penalties & Exit
- Missed contribution penalties:
  * -50 loyalty points
  * Late fee (admin configurable)

- Early exit from pool:
  * Requires admin approval

- Pool completion:

  * Final summary report
  * Option to start new pool with same/different participants

9.5 Investment Tracking (Manual Entry - MVP)

- Admin moderation tools:
  * Mute users
  * Close thread

10.4 Dispute Resolution Tracking

  * Total disputes per month

10.5 External Communication Links

  * WhatsApp group link

13. AD REVENUE SYSTEM & MONETIZATION
================================================================================

13.1 Ad Platform Integration
- Primary: Google AdMob (mobile web)
- Secondary: Google AdSense (desktop web)
- Ad formats:
  * Rewarded video ads (primary for revenue sharing)
  * Interstitial ads (between page transitions)
  * Banner ads (bottom of pages, non-intrusive)

13.2 Ad Viewing Schedule
- Morning session (6 AM - 12 PM):
  * 5 ads available per user
  * Each ad: 15-30 seconds
  * Total time: ~2.5 minutes
- Evening session (6 PM - 11 PM):
  * 5 ads available per user
  * Each ad: 15-30 seconds
  * Total time: ~2.5 minutes
- Daily total: 10 ads per user maximum

13.3 Ad Viewing Mechanism
- "Watch Ads" tab in user dashboard
- Ad availability indicator:
  * Morning: 5 ads available (green badge)
  * Evening: 5 ads available (green badge)
  * Already watched: 0 ads available (gray badge)
- Watching workflow:
  * User clicks "Watch Ads"
  * Ad countdown timer displayed
  * User must watch full ad (no skip for rewarded ads)
  * Ad completion verified
  * Loyalty points awarded (+5 per 10 ads watched)
  * Revenue credited to user account

13.4 Revenue Sharing Model
- Revenue split:
  * Platform: 30%
  * Apartment pool: 70% (distributed among active viewers)
- Individual user earnings:
  * Based on ad completion count
  * Formula: (User's ads watched / Total apartment ads watched) × 70% of revenue
  * Monthly payout calculation
- Earnings dashboard per user:
  * Ads watched today
  * Ads watched this month
  * Estimated earnings this month
  * Total lifetime earnings
  * Payout history

13.5 Revenue Distribution & Withdrawal
- Payout frequency: Monthly
- Minimum withdrawal threshold: ₹100 (or equivalent)
- Payout methods (future - Stripe integration):
  * Bank transfer
  * UPI (India)
  * Mobile money (Africa)
  * PayPal (Global)
- MVP: Manual payout tracking by admin
  * Admin sees total revenue generated
  * Admin calculates per-user share
  * Admin transfers manually
  * Admin marks payout as completed in system

13.6 Ad Viewing Incentives
- Bonus for consistent viewing:
  * 7-day streak: +1.2x earnings multiplier
  * 30-day streak: +1.5x earnings multiplier
  * Entire apartment watches all ads: +10% bonus for all
- Leaderboard: Top ad watchers of the month
- Achievement badges for ad viewing milestones

16. TECHNICAL SPECIFICATIONS
================================================================================

16.1 Frontend (React)
- Framework: React 18+ with Hooks
- Routing: React Router v6
- State Management: 
  * Context API for global state (auth, user profile)
  * React Query for server state caching
- UI Library:
  * Material-UI (MUI) or Ant Design for components
  * TailwindCSS for custom styling
- Form Handling: React Hook Form + Yup validation
- Charts: Recharts or Chart.js
- Calendar: FullCalendar or React Big Calendar
- Image Upload: Cloudinary React SDK
- Real-time Updates: Supabase Realtime subscriptions
- PWA Support: Service workers for offline capability (future)

16.2 Backend (Supabase)
- Authentication: Supabase Auth (email/password)
- Database: PostgreSQL via Supabase
- Real-time: Supabase Realtime for live updates
- Storage: Supabase Storage for backup files (Cloudinary primary)
- API: Supabase auto-generated REST API + PostgREST
- Functions: Supabase Edge Functions (Deno) for:
  * Complex billing calculations
  * Automatic task rotation generation
  * Payment reminder scheduling
  * Loyalty points calculation
  * Revenue sharing calculations
- Row Level Security (RLS) policies:
  * Users can only see data from their apartment network
  * Admins have elevated permissions
  * Private profile data restricted to owner
- Database triggers:
  * Auto-create notifications on certain events
  * Auto-update loyalty points
  * Auto-log user activity

  * profile_photos (300x300, signed upload)

17. USER FLOWS
================================================================================

17.1 New Roommate Onboarding Flow
1. User receives invite link/QR code from admin
2. User clicks link → Registration page
3. Fill personal details (name, age, email, phone)
4. Upload profile photo
5. Upload identity documents (ID card, resume, labour card)
6. Submit application
7. Admin receives notification
8. Admin reviews application
9. Admin approves → User receives welcome email
10. User logs in for first time
11. Tour guide shows key features
12. Admin assigns bed to user
13. User sees first bill and payment due date
14. User explores community meal, tasks, resources

17.2 Monthly Payment Flow
1. 5 days before due date: User receives reminder
2. User opens app → Payment dashboard
3. User sees bill breakdown (rent + utilities + meal)
4. User clicks "Mark as Paid"
5. User enters payment details (method, reference, date)
6. Optional: Upload payment proof screenshot
7. User submits
8. Admin receives verification request
9. Admin checks payment in bank/UPI
10. Admin approves payment
11. User sees "Payment Verified" status
12. User earns +50 loyalty points

17.3 Community Meal Team Grocery Shopping Flow
1. Sunday night: Team B receives notification "Your week starts tomorrow"
2. Monday morning: Team B members see grocery list (based on menu)
3. Team members coordinate (via WhatsApp or dispute chat)
4. Wednesday: Team B goes shopping, buys groceries
5. Team B member 1 uploads receipt photo
6. OCR extracts items and prices
7. Team B member 1 reviews and edits extracted data
8. Submits purchase
9. Admin receives notification
10. Admin reviews items and amount
11. Admin approves
12. System calculates per-person cost
13. All meal participants see their share added to next bill
14. End of week: Other meal participants rate Team B (1-5 stars)
15. Team B sees rating and feedback
16. Team B members earn loyalty points based on rating

17.4 Task Completion Flow
1. Morning 7 AM: User receives notification "Your task today: Clean Kitchen"
2. User acknowledges notification
3. 5 PM: User starts task, clicks "Start Task"
4. User cleans kitchen
5. User takes photo of clean kitchen
6. User uploads photo
7. User clicks "Complete Task"
8. System logs completion time
9. Admin reviews photo (if enabled)
10. Admin approves completion
11. User earns +20 loyalty points
12. User's task completion streak increases

17.5 Resource Booking Flow (Washing Machine)
1. User opens "Resources" tab
2. User selects "Washing Machine"
3. User sees calendar with available slots (green) and booked slots (red/yellow)
4. User selects tomorrow, 2 PM slot (30 minutes)
5. User adds note: "2 loads"
6. User confirms booking
7. User receives confirmation notification
8. Tomorrow 1 PM: User receives reminder "Your washing machine slot in 1 hour"
9. Tomorrow 1:45 PM: User receives reminder "Your slot in 15 minutes"
10. Tomorrow 2 PM: User receives "Your slot has started - Check in now"
11. User goes to washing machine, clicks "Check In"
12. User uses washing machine
13. 2:25 PM: User receives "5 minutes remaining"
14. 2:30 PM: User clicks "Check Out"
15. System releases slot
16. User earns attendance points

17.6 Ad Watching Flow (Morning)
1. Morning 8 AM: User receives notification "5 morning ads available - Earn rewards!"
2. User opens app, clicks "Watch Ads" tab
3. User sees "5 ads available - Estimated earnings: ₹8"
4. User clicks "Start Watching"
5. Rewarded video ad 1 plays (30 seconds)
6. User watches full ad
7. Ad completes, system logs view
8. Automatically loads next ad
9. Repeat for all 5 ads
10. After 5 ads: "You earned ₹8 today! +5 loyalty points"
11. User sees updated earnings in dashboard
12. Evening session repeats same flow

17.7 Dispute Resolution Flow
1. User A has issue with bill amount
2. User A opens "Disputes" section
3. User A clicks "Create Dispute"
4. User A selects "Payment Dispute"
5. User A writes description: "Charged for drinking water but I opted out"
6. User A uploads screenshot of opt-out confirmation
7. User A submits dispute
8. Admin receives notification
9. Admin opens dispute thread
10. Admin reviews evidence
11. Admin checks system records
12. Admin finds User A is correct
13. Admin writes message: "You're right, this was an error"
14. Admin adjusts User A's bill (-₹50)
15. Admin marks dispute as "Resolved"
16. User A receives notification: "Your dispute was resolved"
17. User A sees updated bill amount
